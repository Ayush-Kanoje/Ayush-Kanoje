## 🧠 About Me

I'm an **Electronics & Telecommunication Engineering** student evolving into a **Cloud Security Engineer**. My engineering journey bridges the gap between hardware intuition and cloud-native thinking — from IoT microcontrollers to distributed systems architecture.

```yaml
current_focus:
  role: Cloud Security Engineer (in training)
  learning: [Linux Security, IAM Architecture, Network Defense, Infrastructure as Code]
  methodology: "Build → Break → Secure → Automate → Repeat"
  
philosophy: |
  "Security isn't a feature you add. It's a discipline you practice.
   Every system you build is a system you must be able to defend."
```

**What drives me:**  
I don't just learn tools — I learn *systems thinking*. How distributed architectures scale, how authentication flows break, how automation pipelines fail, and how to harden them. My approach is deliberate, security-first, and constantly evolving.
